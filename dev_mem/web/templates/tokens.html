What we measure: actual context_injected_chars saved per session when the SessionStart hook runs. Divided by 4 to get tokens (standard Claude/GPT approximation).

Manual baseline: what a developer would need to type to re-establish project context from scratch per session.

  • Low (300 tokens) — minimal: stack + current task
  • Mid (600 tokens) — realistic: stack + decisions + recent work
  • High (1200 tokens) — detailed: full project context + preferences + history

Rounds saved: +400 tokens per session — estimated 1–2 clarifying questions Claude would ask without context (~270 tokens each).

Formula: savings = sessions_with_prior × (baseline + rounds_saved) − total_injected

These are estimates. Actual savings depend on how much context you would manually provide and how often Claude needs clarification.
